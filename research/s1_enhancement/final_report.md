# TF Trading S1（日経225 micro/mini）強化検証 最終報告

生成: 2026-05-24 07:39 JST  
対象: 2011-01-04〜2026-05-22 / S1順張り全営業日戦略  
単位: 1u = 既存S1のmicro建玉（夕場/NY一致日は2 micro、逆方向日は1 micro）。損益は税・手数料・slippage前。

## 1. 結論

最終採用候補は **「S1方向は変えず、過去1年比でギャップサイズが80%以上の日だけ取引」**。

- baseline_S1（全営業日）
  - 2011-2026: 3,762取引 / +1,822,227円 / maxDD -99,583円 / 勝率51.9% / MAR 18.30
  - 2021-2026 test: 1,315取引 / +1,376,327円 / maxDD -57,457円 / 勝率54.0% / MAR 23.95
- 採用候補 gap_pr>=0.8
  - 2011-2026: 845取引 / +1,740,471円 / maxDD -22,315円 / 勝率63.4% / MAR 77.99
  - 2021-2026 test: 315取引 / +1,162,533円 / maxDD -18,925円 / 勝率65.7% / MAR 61.43

つまり、取引数は約4分の1になるが、full期間の利益はbaselineの95.5%を残しつつ、DDは77.6%削減。test期間でも利益はbaselineの84.5%を残し、DDを大きく落とせている。

ただし、**gap_pr>=0.8は強すぎる結果なので過剰最適化の警戒は必要**。そのため本番候補は以下の2段階が妥当。

1. **保守採用**: gap_pr>=0.8、1u固定、まずmicroで運用。
2. **攻め採用**: gap_pr>=0.8、利益10万円ごとに+1u、上限3u。増やすのは新高値更新/DD回復後、または月初だけ。

## 2. 採用案

### A案: 主採用 — gap_pr>=0.8 / 1u固定

条件:

- 当日寄り付きギャップ `abs(寄付 - 前日終値)` が、過去252営業日ギャップの80%分位以上。
- 方向は既存S1のまま。ギャップ上なら買い、ギャップ下なら売り。
- NY同方向なら既存通り2 micro、NY逆方向なら1 micro。この既存ルール込みで1u扱い。

採用理由:

- train 2011-2020でも +577,938円 / DD -22,315円 / 勝率62.0%。
- test 2021-2026でも +1,162,533円 / DD -18,925円 / 勝率65.7%。
- rolling 1年・3年・5年でも歴史上は全てプラス。
- 絶対値500円以上などより、分位条件のほうが価格水準変化に強い。

注意点:

- 845取引/15年なので、全営業日S1よりシグナルはかなり減る。
- 2021以降の大きな値幅環境に助けられている可能性はある。
- slippage・寄り付き約定品質を別途検証する必要あり。

### B案: 準採用 — gap_pr>=0.7 / 1u固定

- 2021-2026: 457取引 / +1,285,597円 / DD -28,756円 / 勝率63.2% / MAR 44.71
- full: 1,210取引 / +1,853,535円 / DD -28,756円 / 勝率60.0% / MAR 64.46

gap_pr>=0.8より取引数と総利益が増える。ただしDD/MARは落ちる。実運用で「シグナルが少なすぎる」と感じたら0.7をセカンド候補にする。

### C案: 不採用寄り — abs_gap>=500

- 2021-2026: 84取引 / +823,373円 / DD -18,925円 / 勝率77.4% / MAR 43.51
- train 2011-2020: 1取引 / -4,018円

testだけは強いが、trainにほぼサンプルが無い。価格水準が上がった近年だけに効いている可能性が高く、過剰最適化扱い。

## 4. 資金管理

### 4.1 固定枚数 vs 利益に応じた増し枚

重要: **枚数変更は勝率を上げない**。同じエントリーなら勝ち負けの並びは同じで、変わるのは円損益とDDだけ。

### 4.2 1日最大損失基準 vs 累積DD基準

結論: **1日最大損失だけで枚数を決めるのは危ない。累積DD基準を使うべき。**

| policy | budget | basis | units | test損益 | test DD | full DD |
|---|---:|---|---:|---:|---:|---:|
| baseline_S1 | 100,000 | 1日最大損失基準 | 3u | +4,128,980 | -172,372 | -298,749 |
| baseline_S1 | 100,000 | 累積DD基準 | 1u | +1,376,327 | -57,457 | -99,583 |
| gap_pr>=0.8 | 100,000 | 1日最大損失基準 | 8u | +9,300,265 | -151,398 | -178,522 |
| gap_pr>=0.8 | 100,000 | 累積DD基準 | 4u | +4,650,133 | -75,699 | -89,261 |

1日損失だけを見ると許容枚数が大きくなりすぎる。連敗・損失クラスタで資金を割る可能性があるため、**累積DD + 余裕資金**で見る。

任意の過去開始日から始めた場合の必要資金:

ユーザーの問いへの直接回答:

- 「始めた直後さえ最大DDを食らわなければ勝てるか」  
  baselineは**1年単位ではNo**。rolling 1年で83.2%しかプラスではなく、最悪1年は -39,457円。3年では歴史上ほぼ/完全にプラス。  
  gap_pr>=0.8なら、歴史上はrolling 1年/3年/5年すべてプラスで、1u worst DDも -22,315円。したがって「gap08採用なら過去上はかなりYes。ただし未来保証ではない」。

- 「一日最大DDだけでよいか」  
  **No。累積DD基準が必要。** 1日最大損失基準だとbaseline 100,000円予算で3uを許してしまい、full DDは -298,749円まで膨らむ。

- 「枚数は変えない方が勝率高いか」  
  **枚数では勝率は変わらない。** 勝率を上げるのは取引フィルタ。baseline test勝率54.0%に対し、gap_pr>=0.8は65.7%。fixed_1uでもfixed_3uでも、同じフィルタなら勝率は同じ。

- 「利益を取りに行く増し枚タイミングはどこか」  
  **新高値更新後/DD回復後、利益+10万円ごとに+1u、上限3u。** 実装は月初だけ見直しでもよい。DD中に増やす、連敗後に増やす、1日損失基準で増やすのは非推奨。

## 5. 次の実装案

### 実装ステップ

1. signal生成に `gap_percentile_252` を追加。
   - `abs(today_open - prev_close)` を過去252営業日の絶対ギャップと比較。
   - `gap_percentile_252 >= 0.8` の日のみS1を有効化。
2. dashboard/APIには以下を表示。
   - 今日のgap円幅
   - 過去252日分位
   - S1有効/無効
   - 1u想定枚数（NY同方向なら2 micro、逆なら1 micro）
3. money managementは最初は固定1u。
4. 増し枚モードはfeature flagで別実装。
   - `capital_step_yen=100000`
   - `max_units=3`
   - `sizing_mode=monthly_new_high` または `dd_recovered_new_high`
5. 本番注文はまだ入れない。
   - まずpaper/log運用で、寄り付きデータ取得・判定時刻・約定可能性・slippageを確認。

### 採用ルール案

```text
if gap_percentile_252 >= 0.80:
    trade S1 direction
else:
    no trade

unit = 1
if cumulative_realized_profit_at_month_start >= 100,000: unit = 2
if cumulative_realized_profit_at_month_start >= 200,000: unit = 3
unit = min(unit, 3)
```

より厳密にするなら、増し枚は「過去最高資金更新後」または「DD回復後」に限定。

### 最終推奨

- 明日から検証運用するなら: **gap_pr>=0.8 / 1u固定**。
- 収益を取りに行くなら: **gap_pr>=0.8 / +10万円ごと+1u / 上限3u / 月初または新高値更新後だけ増し枚**。
- 使わない条件: 曜日、連敗停止、VIX単独、MAトレンド単独、前日大陽線/大陰線単独、絶対ギャップ500円以上固定。
